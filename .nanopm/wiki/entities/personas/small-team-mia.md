## Summary

Small-team lead Mia is a technical co-founder or lead developer at a 2–5 person startup with no dedicated PM seat — everyone builds, nobody formally owns "what to build next." This persona is unvalidated and should be treated as a hypothesis: build for Sam first; revisit Mia when there is real inbound pull from small-team users.

## What we know

**Core job to be done**
Coordinate product decisions across a small team without adding a PM layer or forcing everyone into a heavyweight tool. Keep multiple builders oriented toward the same bet.
- "Solo builders are today; small teams are next." — docs/personas.md, 2026-06-26

**The trigger moment**
Standup devolves into prioritization debate. Someone finishes a sprint on something nobody else thinks mattered. The team needs a PM process but won't hire a PM for 18 months.
- "Standup devolves into a prioritization debate. Someone finishes a sprint on something nobody else thinks mattered. The team needs a PM process but won't hire a PM for 18 months." — docs/personas.md, 2026-06-26

**Today's workarounds**
A shared Notion backlog that rots. Slack threads that never close. One person making unilateral calls that others silently disagree with.
- "A shared Notion backlog that rots. Slack threads that never close. One person making unilateral calls that others silently disagree with." — docs/personas.md, 2026-06-26

**How to recognize them (behavioral signals)**
Multiple contributors invoke skills in the same repo. The output gets wired to their existing Linear or GitHub Issues workflow via /pm-breakdown.
- "Multiple contributors invoke skills in the same repo. The output gets wired to their existing Linear or GitHub Issues workflow via /pm-breakdown." — docs/personas.md, 2026-06-26

**Key anxiety**
"A CLI tool — whose terminal runs it? How do we share state?" Shared state across multiple developers is the unsolved problem for this persona.
- "'A CLI tool — whose terminal runs it? How do we share state?'" — docs/personas.md, 2026-06-26

**Confidence**
Assumed — user-stated ("solo builders today, small teams eventually") with zero usage evidence. No small team has ever run nanopm.
- "Not yet validated by any real users — Solo builders are today; small teams are next." — docs/personas.md, 2026-06-26
- "Do not make product decisions to serve Mia until there is pull from actual small-team users." — docs/personas.md, 2026-06-26
