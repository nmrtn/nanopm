## Pitch
Don't change the product — change the recruiting promise. Reposition the cohort ask from "install this and run the pipeline" to a concierge-led "we'll run your first pipeline with you live in 30 minutes on your repo." First-run conversion may be a distribution/onboarding problem, not a product-value one, and concierge is the cheapest way to separate the two before spending any build effort — while guaranteeing more cohort arms actually reach a completed first run.

## Riskiest assumption
That founders who won't self-serve through the pipeline will still feel real value when hand-held — i.e. that abandonment is friction, not a genuine "this isn't worth it" verdict that concierge would only mask.

## Cheapest test
Run 3 live concierge first-pipelines with control-arm founders; if ≥2 voluntarily start a 2nd run within 21 days, friction was the blocker; if 0 do, the value is the problem and no onboarding polish will save it.

## Dissent/tension note
Business: concierge contaminates the form-factor-vs-value test — a hand-held run isn't the self-serve CLI experience the proof quarter is actually trying to measure, so PASS signals may not generalize.
