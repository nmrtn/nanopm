## What's NOT an objective this period

- **Monetization / pricing / revenue model.** *Why tempting:* BUSINESS-MODEL.md names the indefinite deferral as the riskiest assumption — pricing thinking *feels* like progress, and competitor scans (ChatPRD pricing pages) make the absence of a price tag feel unprofessional. *Revisit when:* either proof arm reads PASS — that's BUSINESS-MODEL.md's own falsifiable trigger ("name a revenue model within 30 days"). Charging for a tool no one has yet been proven to voluntarily return to is selling air.

- **New skills (a 22nd or 23rd).** *Why tempting:* every gap surfaced by `/pm-challenge-me` looks fixable by a sharper skill, and shipping a new skill is the cheapest dopamine in this repo. *Revisit when:* a recruited cohort member explicitly names a missing skill as the reason they didn't return. The v0.10.0 Define-first bet shipped 4 new skills before the loop was measured on a single external user — that exact failure mode runs again here on default.

- **Viewer polish beyond what the prototype arm needs to complete a pipeline.** *Why tempting:* the viewer is the most visible surface; polish feels like the product. *Revisit when:* a recruited prototype-arm user names a specific viewer gap that blocked them from running or returning. Anything before that — brand identity refinements, mascot revisions, Memory page upgrades, action-button harmonizations — is what CHALLENGES.md flagged (twice) as the avoidance behavior.

- **A 16th connector / a new ingestion source.** *Why tempting:* "we already have 15, one more is free, and X is the source the next user will need." *Revisit when:* a recruited user names a missing source as the reason they didn't return. The aggregation moat is already at 15 — adding a 16th before the loop retains is "build without learn."

- **Promoting Designer-founder Dani to primary persona.** *Why tempting:* the commit log already designs for Dani; making it official ends the cognitive dissonance. *Revisit when:* both arms read PASS — the cross-matrix per PERSONAS.md's explicit gate. A promotion before the read is a pivot dressed as a discovery.

- **Telemetry / Amplitude / instrumented clone tracking.** *Why tempting:* "if only we could measure who runs what, we'd know." *Revisit when:* qualitative N=10 in either arm fails to produce a readable signal AND a privacy-respecting on-device path exists. STRATEGY.md kept telemetry OFF to avoid repeating the v0.4.0→v0.6.0 premature-infrastructure mistake; solo-fast mode makes small-N DMs valid evidence — that's the bet, don't unbet it on a hunch.

**Action:** Before accepting any new feature request or work item this period, check it against this list. If it matches an anti-goal, the answer is no without a re-prioritization conversation.
