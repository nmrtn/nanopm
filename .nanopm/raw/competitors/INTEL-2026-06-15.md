## Summary

Linear shipped **Coding Sessions** on Jun 11 — the Linear Agent now writes code in-platform using Claude Code and Codex. The structural threat flagged in the 2026-06-10 run ("Linear owns the workflow surface and ships skills natively") is now active product: Linear is the orchestration surface calling into Claude Code, where nanopm runs. The nanopm → Linear handoff is inverting.

ChatPRD added six AI-builder integrations (v0, Granola, Bolt, Lovable, Replit, Cursor) — it is positioning as the PRD/coaching layer atop the AI-app-builder stack, not just the Linear/Notion writing surface.

Discovery during this run surfaced a **new direct philosophical competitor we were not tracking: GitHub Spec Kit** (112k★, MIT, Python). It ships the exact workflow shape nanopm sells — Constitution → Specify → Plan → Tasks → Implement — as multi-agent skill packs for 30+ coding agents including Claude Code. SpecStory (intent-capture layer for AI coding sessions, "Intent is the new source code") is the second adjacent candidate.

**Action:** Add **GitHub Spec Kit** to monitored competitors before the next intel run — it is closer to nanopm's bet than any currently tracked competitor and is ~4,000× larger by GitHub stars. Decide via the question at end of this report. Also: re-read STRATEGY.md against the Jun 11 Linear move; the "/pm-breakdown → Linear" handoff is no longer differentiated.

## Candidate new competitors discovered this run

### GitHub Spec Kit — https://github.com/github/spec-kit

**Why it matters more than any currently monitored competitor:**
- Same workflow shape as nanopm: `/speckit.constitution` → `/speckit.specify` → `/speckit.plan` → `/speckit.tasks` → `/speckit.implement`. nanopm's Define → Discover → Plan → Build is the same loop with different labels.
- Multi-agent support for **30+ AI coding agents** including Claude, Copilot, Gemini — same distribution thesis as nanopm.
- "Skills mode" for agents that support it — explicitly the same artifact format.
- **112,000 stars** (vs. nanopm's 27) — this is the existing 4,000× scale incumbent in the exact philosophical territory.
- MIT license, Python CLI (`specify`). Templates with priority-based local overrides. Constitution system for project principles.

**Differentiation for nanopm:** Spec Kit is engineer-led (engineering teams adopting spec-driven dev). nanopm is PM-led (Define phase = vision/business model/personas BEFORE spec). The "adversarial gate that refuses to ship a non-falsifiable bet" and the typed JSONL state layer are the two things Spec Kit does not have. But Spec Kit owns the surface and the brand.

**Recommended monitored URLs:** repo README, `/specs/templates/`, `/releases`.

### SpecStory — https://specstory.com

**Why adjacent:**
- Tagline "**Intent is the new source code**" — exactly the philosophical territory nanopm claims with the typed memory bet.
- 3,000+ active users; enterprise logos (Uber, Nvidia, SAP, MIT) — past the nanopm cohort scale.
- Captures AI chat history across Cursor / Copilot / Claude Code — same host surfaces as nanopm.
- Auto-generates Cursor rules from captured intent — a wedge that could absorb the "compounding planning state" claim.

**Differentiation for nanopm:** SpecStory captures and replays intent; nanopm structures and refuses bad intent. Adjacent, not direct — worth monitoring at lower cadence.

### Factory.ai (Droid) — https://factory.ai

**Why lower priority:**
- Linear-listed agent partner; "agent-native software development platform" via Droid (CLI + desktop app).
- No explicit PRD/planning/discovery surface on the homepage — coding agent positioning.
- Worth tracking as a coding-agent host, not a direct PM-OS competitor.

## Strategic implications

1. **The Linear handoff is no longer differentiated.** `/pm-breakdown` → Linear was a feature; Linear → write-code-natively is now an end-to-end loop that doesn't need nanopm. STRATEGY.md needs a paragraph on what nanopm does that Linear's Agent + Coding Sessions + Shared Skills + Code Intelligence stack cannot do — and that answer probably has to be Define-phase work (vision/business model/persona/PRD discovery), not breakdown-to-shipped-code. Re-run /pm-strategy with this finding loaded.

2. **GitHub Spec Kit existed all along and we did not see it.** Update the competitive landscape, but more importantly: read Spec Kit's docs carefully and identify what the 112k-star incumbent does NOT do that nanopm's Define phase + adversarial gate + typed state could uniquely own. If the gap is "Spec Kit assumes you already know what to build; nanopm makes you prove it first," that is a clean wedge — but it needs to be claimed in the README, which currently undersells it (per CONTEXT-SUMMARY.md gap).

3. **ChatPRD is now the AI-builder-stack PM layer.** v0/Bolt/Lovable/Replit/Cursor integrations mean ChatPRD targets the prosumer no-code AI-builder user — that is closer to the aspirational "Designer-founder Dani" persona than to "Terminal-native Theo." If Dani is ever promoted to primary persona, ChatPRD is the head-to-head.
