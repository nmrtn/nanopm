## Pitch
Skip the install problem entirely for the recruitment phase: Guillaume runs a 45-minute "concierge install" call with each named KR2 founder, screen-shares the clone+build, then leaves them with a working viewer pointed at a real project of theirs. Treat the first 10 installs as a sales motion, not a product feature. The "install solution" is a Calendly link on the prototype-cohort outreach, not a `.dmg`.

## Riskiest assumption
That a high-touch concierge install converts to a self-driven 2nd run (KR3 ≥ 40%) — i.e. that founders who needed us to install will still come back voluntarily 21 days later without us in the room.

## Cheapest test
Run 2 concierge installs this week with founders already in Guillaume/Nico's network. Measure whether they return to the viewer unprompted within 7 days. If they don't, concierge install is recruitment theater and we need the real one-click path before scaling outreach.

## Dissent/tension note
Business: most strategy-aligned answer — serves the bet by removing install as a variable while we test the actual product — but sidesteps the opportunity as written. We learn nothing about whether a packaged installer would work, and we may need that learning the moment we try to scale past 10.
