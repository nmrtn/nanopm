## What's NOT an objective this period

- **A 18th+ skill** — tempting because every skill run surfaces a "what if we also did X" idea. Cap holds at 17. *Revisit when: ≥3 external users complete a full pipeline AND request the same missing skill by name.*
- **Team features or a web UI** — nobody has asked for multi-user. The CLI is the differentiation. *Revisit when: an external user explicitly requests it more than once.*
- **Distribution infrastructure** (install counters, funnel automation, marketplace listings) — not until validation passes. The riskiest assumption is whether strangers adopt at all, not whether the funnel is optimized. *Revisit when: Objective 1 KR1 is hit.*
- **Polished marketing README** — the ETHOS/builder voice is intentional. *Revisit when: an external user reports it is unreadable.*
- **Monetization** — the business today is adoption and learning, not revenue. *Revisit when: ≥10 strangers have completed the full pipeline unprompted.*
- **New connectors beyond the current 6 targets** — two unprompted requests for the same connector required. *Revisit when: second unprompted request arrives.*

**Action:** Before accepting any new feature request or work item this Q3, check it against this list. If it matches an anti-goal, the answer is no without a re-prioritization conversation.
