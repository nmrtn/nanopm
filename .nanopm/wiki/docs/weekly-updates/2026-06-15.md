# What shipped — week of 2026-06-15

A public changelog covering nanopm releases and improvements from 2026-06-09 to 2026-06-15.

---

## TL;DR

Two minor releases (**v0.9.0** and **v0.10.0**) plus a steady stream of fixes. nanopm now starts with a new **Define phase** that maps your company and product before any planning happens — and every downstream skill reads from a single shared **Context Brief** so the agent doesn't drift between skills. The macOS viewer prototype got a brand identity, a Memory page, and live run streaming.

---

## ✨ New: the Define phase (v0.10.0)

Before you plan, nanopm now establishes ground truth about the company and the product. Four new skills, all dual-mode (reverse-engineer from your code + site, or interview you from scratch):

- **`/pm-vision-mission`** → `VISION-MISSION.md` — mission, vision, values, company stage
- **`/pm-business-model`** → `BUSINESS-MODEL.md` — model, revenue, pricing, GTM motion
- **`/pm-org`** → `ORG.md` — org map, roles, decision-makers, ways of working
- **`/pm-product`** → `PRODUCT.md` — surface area, features, core workflow, technical bets

Every claim in these docs is tagged **Evidenced** or **Assumed**, so you can see at a glance what's grounded vs. inferred. A separate reasoning sidecar at `.nanopm/reasoning/<doc>.md` carries the rationale behind each call.

The retired `/pm-scan` folds into `/pm-product`. The full pipeline is now **Define → Discover → Plan → Build**.

## 🧠 New: shared Context Brief, loaded into every skill

After each Define skill, a subagent regenerates `.nanopm/CONTEXT-SUMMARY.md` — a ~1-page consolidated brief covering what you do, who for, how you make money, why you exist, who decides, and what's not known yet. Every downstream skill (Discover, Plan, Build, Daily Ops) loads it automatically. No more cross-skill drift, no per-skill wiring needed.

## 👥 New skill: `/pm-personas` (v0.9.0)

Define who you're building for — 1–3 JTBD proto-personas plus an explicit **anti-persona** (the tempting user you are deliberately *not* serving). Adaptive: reverse-engineers from code + prior artifacts when they exist, interviews you from scratch when the repo is empty. Wired into the rest of the pipeline.

## 🥊 Renamed: `/pm-audit` → `/pm-challenge-me`

The audit always ended on one adversarial question — that question was the whole point. The skill is now named for it, and it throws three punches per session:

- **strategy** — the question you're avoiding (hard-gated, still required to land)
- **users** — challenges who you think you're serving
- **focus** — challenges where the effort is going vs. the stated goals

Artifact: `.nanopm/CHALLENGES.md` (legacy `AUDIT.md` still read as fallback).

## 📅 New section: Daily Ops

`/pm-challenge-me`, `/pm-standup`, and `/pm-weekly-update` now live in a new **Daily Ops** zone — the skills you run on any given day, outside the planning pipeline. Visible as its own section in the macOS viewer.

## 🗂 Memory system upgrades

- **Per-project config** — each project keeps its own memory; no more global leak across unrelated repos
- **Company-tier docs** — share Define artifacts (`VISION-MISSION.md`, `BUSINESS-MODEL.md`, `ORG.md`) across multiple projects in the same company via symlinks; published automatically on write
- **Fixes:** empty-slug collapse, `find -L` abort on broken symlinks, backup clobber on rerun, dangling company symlinks

## 🖥 macOS viewer prototype

A read-only SwiftUI viewer for nanopm artifacts (still labeled a throwaway prototype):

- **Nano brand identity** — colors, type, mascot, Claude-style loaders
- **Memory page** — replaces the generic "Other" bucket
- **Live activity monitor** — stream parallel runs, follow conversations in flight
- **Interactive runs** — in-app questions, model transcript, session resume
- **User context on launch** — pass optional context when starting a skill
- **Per-phase Overview pages** — Discover, Plan, Build each get a landing card
- **Define renders as the first phase** with a Context Brief card up front
- **Discover** — Run button on every skill; Competitors browser with report history; designed summary page
- **App icon** — paper-cut lighthouse
- **Fix:** no longer crashes when launched outside a `.app` bundle (PR #30)
- **Fix:** friendly auth-failure messaging; consolidated run diagnostics
- **Safety:** dropped `bypassPermissions`, scoped tools, honest docs on option-2 runs

## 🛠 Other fixes

- `nanopm_context_append` no longer fails under non-UTF-8 shells (em-dashes in mission statements were tripping `LC_ALL=C` runs)
- `setup` reads version from the `VERSION` file instead of a hardcoded literal
- `pm-run` option B no longer halts after the audit step
- Define skills now branch on a single fact (does the target doc exist?) instead of sniffing the surrounding evidence — refine vs. create modes are explicit
- Cross-document context in Define skills is gathered by a retrieval subagent that returns only relevant slices, not raw dumps
