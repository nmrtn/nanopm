## Strategic implications

**deanpeters:** The MARKETPLACE_STRATEGY.md and live Streamlit UI are the biggest signal. They're thinking about reaching users beyond GitHub — which is nanopm's current distribution path too. With 60+ skills and community contributions accelerating, they're going wide. But wide is their weakness: 60 isolated skills with no pipeline, no memory, no chaining. nanopm's compound pipeline (audit → strategy → PRD → each skill reading the last) is indefensible by a flat skill collection. Lean into that explicitly in the README and any distribution copy.

**gstack:** GBrain is the one to watch. Persistent cross-session memory is exactly what nanopm does at the PM layer — gstack now does it at the eng layer. They're complementary today (nanopm ends where OpenSpec/gstack begins), but if gstack ever ships PM skills backed by GBrain, the differentiation collapses fast. Keep the OpenSpec integration strong and make the nanopm ↔ gstack handoff story crisp.

**prawduct:** Removing the CLI while expanding templates into human-interface and unattended-operation suggests a scope expansion toward full product governance (not just code quality). Less direct competition with nanopm today, but worth watching — if they add PM discovery/strategy skills on top of their critic infrastructure, they'd be building toward the same user.
