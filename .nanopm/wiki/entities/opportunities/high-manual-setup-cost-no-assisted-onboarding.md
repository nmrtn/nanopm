## 1. Problem summary
Getting nanopm to a useful state has a heavy upfront cost: defining company context, personas, business model, etc. is largely manual and takes roughly an hour before the loop pays off. That start cost is front-loaded onto a new user who hasn't yet seen value (compounding the first-run-value problem). The founder's fix: an assisted onboarding that pulls existing context from where it already lives — "you want to fetch from Notion? connect Notion, boom, it grabs the info and proposes drafts" — turning an hour of manual Q&A into a connect-and-confirm flow.

## 2. Value to the user
### Job to be done
A new user wants to stand up their company/product context fast by importing what they already have (Notion, site, docs) and confirming drafts, rather than answering an hour of setup questions. The alternative today is a long manual interview per Define skill.

### Where we fall short
**Setup is ~1h and very manual; no connector-assisted import**
The company-context phase has many steps and is manual; there's no onboarding that ingests existing sources to pre-fill it.
- "On a un gros coût de démarrage de setup. C'est long de faire toute la partie contexte de boîte, personas, etc. — au moins une heure quasi de setup parce que c'est très manuel. Si on avait un onboarding bien foutu : tu veux aller chercher dans Notion ? connecte Notion, boom il récupère les infos et te propose des trucs." — Granola call w/ Nico, 2026-06-30 (raw/interviews/2972bcf0bb4c.md)

## 3. Value to the company
Setup cost is a direct adoption tax and a likely drop-off point before the first end-to-end run — the moment the value bet most needs to land fast. Guardrail: internal founder signal (N=1, two co-founders).
