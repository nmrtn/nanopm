# Vision & Mission
Mode: reverse-engineered from codebase + README + maintainer confirmation · 2026-06-25

## Mission (today)
Replace the PM workflow end-to-end for AI coding agents. Give a solo builder an
autonomous PM that runs the whole product loop — discover → decide → plan → build →
learn — in the same terminal as their code, so the person who is both PM and engineer
never has to leave their environment to do product thinking.

## Vision (3–5 years)
The PM layer for agentic development. Anyone building with an AI coding agent
(Claude Code, Vibe, Codex) plans, decides, and ships product through nanopm, on a
memory that **compounds** across sessions instead of starting cold each time.

## Core values
- **Honesty over comfort.** Every claim is tagged Evidenced vs Assumed. The job is to
  sharpen the user's thinking, not validate it.
- **Name the question you're avoiding.** Adversarial by design — the challenge gate, the
  falsifiable bet, the anti-goals.
- **Subtract before you add.** The no's are as strategic as the yes's; a roadmap that
  says no is more valuable than one that says yes to everything.
- **Ship, then learn.** Real signal only comes from real use. A crappy v1 that ships
  beats a perfect one that doesn't.
- **Single source of truth.** One wiki the agent owns and maintains, not scattered docs.

## Company stage
Early / pre-product-market-fit. Solo founder + 1 collaborator. **Zero external users.**
Still validating whether the core loop is even wanted — the product works mechanically,
but its value is unproven in real use.

## The one belief everything rests on
That solo builders doing PM **in their terminal** actually want an autonomous,
compounding PM loop — rather than doing product thinking in Notion / ChatGPT / their
head. Everything else follows from this. It is currently **unproven**.
