## Pitch
Don't ship an installer at all this cycle — instead, write a 1-page "Prototype Cohort Onboarding" doc (Notion or the marketing site) that links to a pre-built unsigned `.dmg` behind a Typeform gate ("name, role, project you'd run this on"). The gate qualifies for KR2's "non-terminal-native" criterion before we spend a build minute. Recruitment, not installation, becomes the artifact this opportunity ships.

## Riskiest assumption
That the binding constraint on KR2 = 0 is qualified-lead supply, not install friction — i.e. that if 10 qualified non-terminal founders walked up tomorrow, we already have enough install scaffolding to land them.

## Cheapest test
Post the Typeform-gated pitch in 3 founder communities (IndieHackers, a designer-founder Slack, one Discord) this week. If <5 qualified responses arrive in 7 days, the bottleneck is recruitment/positioning, not install — and we should redirect this slot to the Roadmap NEXT item ("Prototype cohort recruitment via the Discovery DB pitch") rather than building installer plumbing.

## Dissent/tension note
Business: this solution refuses the opportunity's framing. It says "install isn't the problem; recruitment is" — which contradicts the opportunity's stated value-to-the-company. If install really is the blocker, this option leaves it untouched and we'll discover that the hard way after burning a recruitment push on a broken funnel.
