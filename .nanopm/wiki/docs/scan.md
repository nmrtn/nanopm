## What This Product Actually Does

nanopm is a bash runtime + set of markdown prompt files (SKILL.md) that installs slash commands into Claude Code, each implementing one phase of the PM planning cycle. It writes markdown artifacts to `.nanopm/` and threads context between skills via an append-only JSONL memory at `~/.nanopm/memory/{slug}.jsonl`. The LLM (Claude) is the execution engine — the SKILL.md files are structured prompts, not code.

README claim matches code reality closely. The main divergence: README positions this as a "PM layer" for teams; the code has no multi-user concepts, no auth, no roles — it is explicitly single-developer tooling.

## Who It's Actually Built For

Inferred from code: solo developers running Claude Code, with a git repo, python3, and bash available. No team constructs exist anywhere in the codebase (no user auth, no shared memory, no roles). The memory model is per-machine, per-project — not shareable.

6. **PRD + tickets** — `/pm-prd` then `/pm-breakdown` writes spec and creates tickets in Linear/GitHub

## Technical Bets

- **SKILL.md as execution engine** — the entire product depends on Claude following multi-phase markdown instructions reliably. If Claude skips a phase, hallucinates a section, or fails to call the right tool, the artifact is wrong with no error signal. No guardrails exist for silent failure.
- **Bash-only runtime** — lib/nanopm.sh is pure bash (no node, no python except for JSON formatting). Durable and portable, but limits what the runtime can do — no structured output validation, no schema enforcement on artifacts.
- **JSONL append-only memory, no compaction** — `nanopm_context_all` returns everything unfiltered as the project grows. Long-lived projects will accumulate noise. No pruning, search, or summarization is implemented.
- **No web UI** — everything is terminal + markdown files. Correct bet for Claude Code users today; fragile if the audience broadens.

The syntax test validates structure but not data flow. A broken `nanopm_context_append` call in any skill would silently break the compounding without any test catching it.
