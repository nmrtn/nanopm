## NEXT (1-3 months out)

*Directionally committed. Will be refined as NOW items ship.*

- **Run the location assumption test** — DM 10 Claude Code users: "where do you do your planning thinking — terminal or somewhere else?" If 7+ say elsewhere as a habit, reframe product from "plan in terminal" to "generate artifacts in terminal, use them anywhere." → ties to Strategy: cheapest test
- **Distribution pass** — post nanopm in Claude Code community (Discord/GitHub discussions), indie hacker forums, and X/Twitter; track install-to-pipeline-run conversion → ties to Obj 1: KR1, KR2
- **Frictionless install** — reduce time-to-first-pipeline from clone → run to under 3 minutes; document the exact steps for a stranger → ties to Obj 1: KR2 (conversion rate)

## LATER (3+ months / when we get there)

*Good ideas that aren't the priority now.*

- **Connector integrations (GitHub/Linear)** — auto-populate audit Q4 from merged PRs and backlog; only worth building after artifact quality is validated and there are users whose context is worth pulling
- **pm-discovery skill enhancement** — stronger opportunity mapping; useful but not on the critical path to the Q2 objectives
- **Output format options** — export to Notion, Linear, or markdown per-user preference; premature without knowing where users actually work

---

## Explicitly NOT on the roadmap

- **Not a SaaS or web UI** — intentional ecosystem decision; revisit only if CLI distribution proves structurally impossible
- **Not new skills** — 9 skills is enough surface area; adding before validating compounds unvalidated complexity
- **Not community building (yet)** — no point building an audience before the artifact quality problem is solved
