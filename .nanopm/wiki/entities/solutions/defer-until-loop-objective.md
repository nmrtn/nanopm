## Pitch
Build nothing now; put traceability on the Obj3 backlog with an explicit gate — start only after ≥2 external users return for a second session. Traceability is "the spine that lets the loop later learn from outcomes," which is literally Obj3's "context compounds" thesis, and Obj3 is sequenced after July's demand-validation KR. Building it now is optimization without a baseline, against an open anti-goal. The disciplined answer: distribution before features.

## Riskiest assumption
That the broken thread doesn't bite during the July validation itself — that a stranger can complete a full pipeline without the why→spec→metric link, so deferring costs nothing pre-PMF.

## Cheapest test
Run one full pipeline end-to-end as a stranger would (fresh project, no maintainer memory) and observe whether the missing thread actually blocks completion or merely degrades it — if it blocks, it's a July problem, not an Obj3 one.

## Dissent/tension note
Business: the disciplined answer to the anti-goal — but it bets the thread won't bite during the very validation it's protecting; if a stranger abandons because "why am I building this" is unanswerable, we mis-shelved a July blocker.
