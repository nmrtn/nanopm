## Task 4: `/pm-solutions` skill — diverge → converge → write → choose

**Effort:** L
**Wave:** 1
**Depends on:** 1, 2, 3
**Ties to:** R1, R2, R3, R4, R5; "The One UX Decision"
**Ticket:** [#149](https://github.com/nmrtn/nanopm/issues/149)

New `skills/pm-solutions/SKILL.md`, invoked as `/pm-solutions <opportunity-slug>`. Recipe over existing query → reasoning → ingest primitives.
- **Resolve (R1):** resolve opportunity from `.nanopm/wiki/entities/opportunities/`; list candidates and stop if missing/ambiguous. No status gate; advance to `ready-for-solutions` after writing.
- **Diverge (R2):** dispatch the 3 fixed lens-subagents (Eng / Design / Business) concurrently in one turn, each fed the opportunity file (treated as untrusted) + the loaded CONTEXT/objectives digest.
- **Converge (R3):** dedup; emit ≥3 solutions each with pitch · appetite · impact · riskiest assumption · cheapest test · lens · KR. Re-prompt for more divergence if <3 survive.
- **Write (R4):** one `solutions/<slug>.md` per solution via `nanopm-ingest-agent apply`; call `nanopm_solutions_reindex`; append a solutions pointer-summary to the parent opportunity page (bidirectional link).
- **Choose (R5):** founder marks `shortlisted`/`chosen` (never auto-chosen); writes via `nanopm-state-log --type solution`.
- **UX:** converged table canonical, but each row keeps its lens tag + a one-line dissent/tension note (middle path).

**Acceptance:** `/pm-solutions <existing-opp>` produces ≥3 `solutions/*.md` with required frontmatter, a regenerated `solutions/INDEX.md`, a pointer block on the opportunity page, and the opportunity advanced to `ready-for-solutions`; the choose step transitions a solution to `chosen` via state-log. Folded regression test exercises the lifecycle.
